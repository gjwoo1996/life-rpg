# Life RPG 사용 방법

## 0. 로컬에서 UI 확인 (개발자)

exe를 빌드하기 전에 로컬에서 앱을 띄워 UI와 동작을 확인할 수 있습니다.

- **필요 환경**: Node.js(LTS), Rust(stable). Windows에서 exe를 만들 경우 Visual Studio Build Tools 등이 필요할 수 있습니다.
- **명령**:
  ```bash
  npm install
  npm run tauri:dev
  ```
- **동작**: Tauri 개발 모드로 창이 열리고, 프론트엔드는 HMR로 갱신됩니다. 실제 exe와 동일한 UI를 로컬 창에서 확인할 수 있습니다.
- **참고**: WSL2 안에서는 GUI가 필요하므로, **Windows 호스트**에서 터미널을 열고 위 명령을 실행하는 것을 권장합니다.
- **빌드 흐름**: 로컬에서 UI 확인 → 만족하면 커밋 후 `main`에 push → GitHub **Actions** 탭에서 **Build Windows** 워크플로 실행(또는 push 시 자동 실행) → 완료 후 **Artifacts**에서 exe·설치 파일 다운로드.

---

## 1. 준비

### exe 받기
- GitHub 저장소 → **Actions** 탭 → **Build Windows** 워크플로 실행 후 완료 시 **Artifacts**에서 `life-rpg-windows` 다운로드.
- 압축 해제 후 `life-rpg.exe` 실행 (또는 `nsis` 폴더 안 설치 프로그램으로 설치 후 실행).

### AI 분석을 쓸 경우 (선택)
- WSL2에서 Ollama가 떠 있어야 함 (예: DevContainer에서 `docker compose up` 후 Ollama 11434 포트 사용).
- Windows에서 exe 실행 시 **같은 PC**에서 WSL2가 켜져 있으면 `http://localhost:11434`로 자동 연결됨.

---

## 2. 실행

- **life-rpg.exe** 더블클릭.
- 또는 설치 프로그램(NSIS)으로 설치한 경우 시작 메뉴/바탕화면에서 **Life RPG** 실행.

---

## 3. 처음 사용 (캐릭터 만들기)

1. 앱이 켜지면 **캐릭터 이름** 입력.
2. **캐릭터 생성** 클릭.
3. 이후부터는 메인 화면(대시보드)이 나옴.

---

## 4. 매일 쓰는 방법

### 목표
- **목표 이름**, **시작일**, **종료일**, **연결할 능력** 선택 후 **목표 추가**.

### 활동 기록
- **날짜**, **오늘 한 일** 입력.
- **획득 XP**: 직접 숫자 입력하거나, **AI 분석** 버튼으로 자동 계산(아래 5번).
- **기록 저장** 클릭 시 XP가 반영되고 캐릭터 레벨이 올라감.

### 캐릭터
- 상단에 **이름**, **레벨**, **XP**, **능력치**가 표시됨.

---

## 5. AI 분석 쓰는 경우

1. **WSL2에서 Ollama 실행** (예: `OLLAMA_HOST` 사용 중이면 해당 서비스가 11434에서 동작).
2. 앱에서 **활동 내용** 입력 후 **AI 분석** 클릭.
3. 분석이 끝나면 **획득 XP** 칸이 자동으로 채워짐 → 필요 시 수정 후 **기록 저장**.

---

## 요약

| 하려는 것       | 하는 일 |
|----------------|----------|
| exe 받기       | GitHub Actions → Build Windows → Artifacts에서 다운로드 |
| 실행           | life-rpg.exe 더블클릭 (또는 설치 후 실행) |
| 캐릭터 만들기  | 이름 입력 → 캐릭터 생성 |
| 목표 추가      | 목표 이름·기간·능력 선택 → 목표 추가 |
| 활동 기록      | 날짜·내용·XP 입력(또는 AI 분석) → 기록 저장 |
| AI로 XP 계산   | WSL2에서 Ollama 실행 → 활동 입력 후 AI 분석 클릭 |
