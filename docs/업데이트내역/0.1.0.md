# 업데이트 내역

## 0.1.0

**기준일**: 2025-03-12  
**개요**: 홈 툴팁 위치, 캘린더 색상 가시성, 활동 요약 UI 제거, LLM 한국어 프롬프트, 일본어 모델 제거, 로그 뷰어 3종 탭

### 1. 홈 화면 경험치 프로그레스바 툴팁

- **변경**: 마우스/포커스 시 툴팁이 **프로그레스바 아래**가 아니라 **바 안(오버레이)**에 표시되도록 수정
- **효과**: 카드가 세로로 늘어나지 않음 (레이아웃 시프트 제거)
- **파일**: `src/components/CharacterView.tsx`

---

### 2. 캘린더 목표 색상 가시성

- **변경**: 활동 기록 캘린더 날짜 셀 내 목표 색상 점 크기 확대 (`w-1.5 h-1.5` → `w-3 h-3`), `ring`/`border`로 경계 추가
- **효과**: 밝은 배경에서도 목표 색상이 더 잘 보임
- **파일**: `src/pages/ActivityPage.tsx`

---

### 3. 활동기록에서 요약 미노출

- **변경**: `summary`는 분석/LLM 입력용으로만 사용하고, 사용자 UI에는 노출하지 않음
- **제거**: 활동 기록 캘린더 날짜 셀의 요약 문구, 해당 날짜 상세(ActivityDayPage)의 "요약: ..." 블록
- **유지**: 백엔드 `activity_logs.summary` 저장·일일/목표 분석에서의 사용 로직은 그대로 유지
- **파일**: `src/pages/ActivityDayPage.tsx`, `src/pages/ActivityPage.tsx`

---

### 4. LLM 요약/분석 응답 한국어 지정

- **변경**: 요약·일일 분석·목표 분석 프롬프트에 "한국어로 답하라"는 지시 명시
- **파일**: `src-tauri/src/ai/prompt.rs`

  - `build_daily_analysis_prompt`: "Reply in Korean."
  - `build_goal_analysis_prompt`: "Reply in Korean."

### 5. 일본어 LLM 모델 및 연동 제거

- **코드**: `src-tauri/src/ai/client.rs`에서 `japanese_model()`, `is_japanese()` 제거, 모든 LLM 호출이 기본 모델만 사용
- **스크립트**: `.devcontainer/scripts/setup-ollama.sh`, `ensure-ollama-model.sh`에서 일본어 모델 pull 제거
- **도커**: `ensure-ollama-model.sh`에 이미 설치된 일본어 모델 삭제 단계 추가 (`ollama rm schroneko/gemma-2-2b-jpn-it:q4_0`)
- **문서**: `docs/업데이트내역/0.0.0.md`에 일본어 모델 제거 반영

---

### 6. 프로그램 로그 3종 탭

- **기능**: 로그 뷰어에서 **Rust 처리** / **LLM** / **Ollama** 탭으로 구분해 각각 확인 가능
- **Rust**: 일반 앱 로그 (`log::info!` 등) → `tauri_plugin_log`로 프론트 전달
- **LLM**: 요약·분석·XP 분석 등 LLM 호출 이벤트 (메시지 접두어 `[LLM]`)
- **Ollama**: Ollama API 요청/응답 (URL, 모델, 상태, 에러) (메시지 접두어 `[Ollama]`)
- **구현**: `logStore`에 `source` 필드 추가(메시지 접두어로 분류), LogViewer에 탭 UI 및 탭별 필터
- **로그 레벨**: `LevelFilter::Debug`로 설정해 더 많은 로그 수집
- **파일**: `src-tauri/src/ai/client.rs`, `src-tauri/src/lib.rs`, `src/stores/logStore.ts`, `src/components/LogViewer.tsx`
