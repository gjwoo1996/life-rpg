# 업데이트 내역

## 0.0.0

**기준일**: 2025-03-10  
**개요**: 요구사항 개선사항 반영 — GitAction 수동·버전, 사용자 능력·목표 색상, 활동기록 오늘만·요약·2depth 화면, 일일/누적 분석, 사용자 분석 메뉴, 일본어 LLM 분기

---

### 1. GitAction

- **트리거 변경**: `push` 제거, **수동 실행만** (`workflow_dispatch`) 유지
- **버전 입력**: 워크플로 입력 `version` 추가 (기본값 `0.1.0`)
- **빌드 반영**: 빌드 전 `tauri.conf.json`의 `version`을 입력값으로 갱신
- **아티팩트**: 업로드 이름에 버전 포함 (`life-rpg-windows-v{version}`)

---

### 2. 목표 (Goals)

- **캘린더 색상**: `goals` 테이블에 `calendar_color` 컬럼 추가 (기본 `#6366f1`)
- **UI**: 목표 추가 시 컬러 피커·프리셋 색상 선택, 목표 목록에 색상 표시

---

### 3. 사용자 능력 (Abilities)

- **고정 5스킬 제거**: 기존 `stats` 5종 대신 **사용자 정의 능력**으로 전환
- **데이터**: `abilities`(캐릭터별 능력 이름), `ability_stats`(능력별 XP 0~100, Lv 1~10)
- **목표 연동**: 목표 추가 시 능력 선택(기존 목록) 또는 **직접 입력** (combobox 형태)
- **활동 저장**: 경험치는 사용자 입력 없이 **AI가 분석해 부여**, 해당 능력의 `ability_stats`만 갱신
- **UI**: 캐릭터 화면에 능력별 **가로 퍼센트 바** 표시, focus 시 총/현재 경험치 툴팁

---

### 4. AI 분석

- **능력 기반**: `analyze_activity(content, ability_names)` — 요청한 능력 이름에만 XP 배분
- **모델 설정**: 환경변수로 유연하게 변경
  - `LIFERPG_LLM_MODEL_DEFAULT`: 기본 분석용 (기본값 `qwen2.5:7b`)
  - `LIFERPG_LLM_MODEL_JA`: 일본어용 (기본값 `schroneko/gemma-2-2b-jpn-it:q4_0`)
- **일본어 분기**: 내용에 히라가나·가타카나·한자가 있으면 일본어 전용 모델 사용 (요약·일일 분석·활동 XP 분석)

---

### 5. 활동기록 (Activity Logs)

- **날짜 제한**: **오늘 날짜만** 기록 가능 (프론트 고정, 백엔드 검증)
- **요약**: 저장 시 LLM으로 활동 내용 요약 후 `activity_logs.summary`에 저장
- **경험치**: 사용자 입력 제거, 저장 시 서버에서 AI 분석 후 능력별 XP 자동 반영
- **라우팅**: 활동기록 전용 화면으로 분리
  - **1depth**: 월별 **캘린더** — 목표 색상·해당 날짜 활동 요약 표시
  - **2depth**: 날짜 클릭 시 **해당 일 로그 목록** + **일일 분석** 표시
- **일일 분석**: 활동 저장 시 해당 날짜 로그를 모아 LLM 분석 후 `daily_analyses` 테이블에 저장

---

### 6. 일일/누적 분석 데이터

- **daily_analyses**: `(date, character_id, analysis_text)` — 날짜별 일일 분석
- **goal_analyses**: `(goal_id, character_id, summarized_context, last_analyzed_at, analysis_text)` — 목표별 누적 분석용 테이블 및 `get_goal_analysis(goal_id)` 명령 추가 (누적 분석 갱신 로직은 추후 확장 가능)

---

### 7. 신규 메뉴 — 사용자 분석

- **경로**: `/analysis`
- **기능**: 등록된 목표별로 `goal_analyses` 조회 후 분석 문구 표시
- **일본어**: 목표/활동이 일본어일 때 위 환경변수 일본어 모델로 분석 분기 적용

---

### 8. 기타

- **라우팅**: React Router (HashRouter) 도입 — `/`(홈), `/activity`, `/activity/:date`, `/analysis`
- **네비게이션**: 상단에 홈 / 활동기록 / 사용자 분석 링크 추가
- **홈**: 목표·오늘 활동 기록 입력 + "활동 캘린더 보기" 링크로 활동 화면 이동

### 9. 정리 및 환경

- **stats 테이블 제거**: 고정 5스킬용 `stats` 테이블 및 `get_stats` 명령 제거. 기존 DB는 마이그레이션 시 `DROP TABLE IF EXISTS stats` 적용.
- **일본어 모델 설치**: devcontainer용 Ollama 스크립트(`setup-ollama.sh`, `ensure-ollama-model.sh`)에 일본어용 모델 `schroneko/gemma-2-2b-jpn-it:q4_0` 설치 단계 추가. 환경변수 `LIFERPG_LLM_MODEL_JA`로 변경 가능.
